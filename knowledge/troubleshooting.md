# 常见故障排查指南

> 最后更新：2026-02-26
> 维护人：技术支持团队
> 说明：本文档包含用户最常遇到的技术问题及排查步骤。客服回答故障类问题时以本文档为准。

---

## 一、登录相关问题

### 问题：无法登录 / 登录失败

**可能原因与排查步骤**：

1. **密码错误**
   - 确认是否开启了大写锁定（Caps Lock）
   - 尝试使用「忘记密码」重置密码
   - 连续输错 5 次会锁定 30 分钟

2. **账号不存在**
   - 确认注册时使用的是邮箱还是手机号
   - 检查是否拼写错误（常见：.com 写成 .cn）
   - 可能使用了微信一键登录，尝试微信扫码

3. **网络问题**
   - 切换 Wi-Fi 和移动数据试试
   - 检查是否使用了 VPN 或代理（部分地区需关闭）
   - 清除浏览器缓存后重试

4. **账号被冻结**
   - 异常登录触发安全机制 → 转人工客服解冻

**标准话术**：「请您先检查一下密码是否正确，注意大小写。如果忘记了密码，可以点击登录页的"忘记密码"重置。如果问题仍然存在，我帮您转接技术支持。」

---

### 问题：验证码收不到

**排查步骤**：

1. 检查手机号是否正确（注意区号）
2. 查看短信是否被拦截到垃圾短信箱
3. 确认手机信号正常
4. 60 秒后可重新发送
5. 连续发送 3 次仍收不到 → 尝试语音验证码
6. 语音验证码也收不到 → 转人工客服

**标准话术**：「验证码一般 1 分钟内送达，请检查一下是否在短信垃圾箱里。您也可以 60 秒后点击重新发送。如果多次尝试仍收不到，我帮您转接技术支持。」

## 二、页面与显示问题

### 问题：页面加载缓慢或白屏

**排查步骤**：

1. **检查网络**：尝试访问其他网站确认网络正常
2. **清除缓存**：
   - Chrome：设置 → 隐私与安全 → 清除浏览数据
   - Safari：偏好设置 → 隐私 → 管理网站数据 → 全部移除
   - App：设置 → 存储 → 清除缓存
3. **更换浏览器**：尝试使用 Chrome 最新版
4. **关闭浏览器扩展**：部分广告拦截器会影响页面加载
5. **检查状态页**：访问 status.example.com 确认服务是否正常

**如果以上都无效**：收集以下信息后转技术支持：
- 浏览器版本
- 操作系统版本
- 具体报错截图
- 问题出现的时间和频率

---

### 问题：App 闪退 / 崩溃

**排查步骤**：

1. **更新 App**：确认已安装最新版本
   - iOS：App Store → 搜索产品名 → 更新
   - Android：应用商店 → 我的应用 → 更新
2. **重启 App**：彻底关闭后重新打开
3. **重启手机**：部分内存问题重启可解决
4. **检查存储空间**：手机存储不足可能导致崩溃（建议至少保留 500MB）
5. **卸载重装**：备份数据后卸载重装（云端数据不会丢失）

**标准话术**：「建议您先确认 App 是否是最新版本，可以到应用商店检查更新。如果已经是最新版，尝试卸载重装试试，您的数据都保存在云端不会丢失 😊」

## 三、功能使用问题

### 问题：数据导入失败

**排查步骤**：

1. **文件格式**：仅支持 .csv、.xlsx、.xls 格式
2. **文件大小**：单次导入不超过 10MB（约 5 万行）
3. **编码问题**：CSV 文件请使用 UTF-8 编码保存
4. **表头格式**：第一行必须是列标题，不能有空行
5. **必填字段**：检查是否缺少必填列（如客户名称、联系方式）
6. **特殊字符**：去除单元格中的换行符和特殊符号

**导入模板下载**：登录后进入「设置 → 数据管理 → 下载导入模板」

**标准话术**：「导入失败通常是文件格式问题。请确认文件是 .csv 或 .xlsx 格式，文件大小不超过 10MB，并且第一行是列标题。您可以下载我们的标准导入模板来填写数据。」

---

### 问题：通知/提醒没有收到

**排查步骤**：

1. **检查通知设置**：「设置 → 通知管理」确认相关通知已开启
2. **检查手机权限**：
   - iOS：设置 → 通知 → [产品名] → 允许通知
   - Android：设置 → 应用 → [产品名] → 通知 → 开启
3. **检查免打扰模式**：确认手机未开启勿扰模式
4. **邮件通知**：检查垃圾邮件箱，将我们的发件地址加入白名单
5. **确认订阅状态**：免费版部分通知功能有限制

## 四、支付相关问题

### 问题：付款成功但服务未开通

**排查步骤**：

1. **等待 5 分钟**：系统处理有时有延迟
2. **刷新页面或重启 App**
3. **检查订单状态**：「我的订单」中确认订单状态是否为"已完成"
4. **检查支付记录**：确认扣款是否成功（查看银行/支付宝/微信账单）
5. **如果确认已扣款但未开通** → 转人工客服，提供：
   - 订单号
   - 支付截图
   - 支付时间

**标准话术**：「请您稍等几分钟，系统可能有短暂延迟。如果 5 分钟后仍未开通，请您提供一下订单号和支付截图，我帮您核实。」

---

### 问题：重复扣款

**处理方式**：此问题必须转人工客服处理。

**收集信息**：
- 扣款时间和金额
- 支付账单截图
- 订单号

**标准话术**：「非常抱歉给您带来不便！这个问题我帮您转接专业同事来处理，请您提供一下扣款截图和订单号，我们会尽快核实并退回多扣的款项 🙏」

## 五、常见错误码说明

| 错误码 | 含义 | 用户可见说明 | 处理方式 |
|-------|------|------------|---------|
| 400 | 请求参数错误 | "操作失败，请检查填写的信息" | 引导用户检查输入 |
| 401 | 未认证 | "登录已过期，请重新登录" | 引导用户重新登录 |
| 403 | 权限不足 | "您没有权限执行此操作" | 确认用户的套餐/角色 |
| 404 | 资源不存在 | "您访问的页面不存在" | 引导用户检查链接 |
| 429 | 请求过于频繁 | "操作太频繁，请稍后再试" | 等待 1 分钟后重试 |
| 500 | 服务器内部错误 | "系统开小差了，请稍后再试" | 如持续出现转技术支持 |
| 502/503 | 服务暂时不可用 | "系统维护中，请稍后再试" | 查看 status.example.com |
